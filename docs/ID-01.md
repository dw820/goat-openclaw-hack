# ID-01: ERC-8004 Provider Identity Setup

**Module**: On-chain provider identity
**Time estimate**: 10 min
**Priority**: Must-Have
**Dependencies**: None

---

## Scope

Register the **provider's** ERC-8004 identity on GOAT Testnet3 via `@goathackbot`. Each provider who wants to offer paid inference registers their own on-chain identity. The resulting `agentId` is included when the provider registers on the marketplace, enabling clients to verify the provider before paying.

**Outputs**:
- `agentId` (ERC-721 NFT on chain 48816) — belongs to the provider
- x402 merchant credentials (API key, secret, merchant ID) — used in the provider sidecar
- Test tokens (USDC, USDT, gas) in provider's wallet
- Provider stores these creds in `provider-sidecar/.env`

## Prerequisites

- A Testnet3 wallet address (`0x...`) — use MetaMask or any EVM wallet
- Access to `@goathackbot` on the hackathon Discord/Telegram

## Steps

### Step 1: Prepare Registration Info

Gather the following before messaging the bot:

| Field | Value |
|-------|-------|
| **Project name** | A name for your provider (e.g., `alice-llama3-provider`) |
| **Wallet address** | Your `0x...` address |
| **Description** | "Inference provider — serving LLM endpoints on the Decentralized Inference Marketplace via x402 payments" |

### Step 2: DM @goathackbot

Send the bot your project name, wallet address, and description.

**@goathackbot will**:
1. Create your x402 merchant account
2. Register your ERC-8004 agent identity on Testnet3
3. Send test USDC, test USDT, and gas tokens to your wallet
4. Return a `.env` block with credentials

### Step 3: Save Credentials

You will receive something like:

```env
# x402 Merchant Credentials
GOATX402_API_URL=https://x402-api-lx58aabp0r.testnet3.goat.network
GOATX402_MERCHANT_ID=alice-llama3-provider
GOATX402_API_KEY=gx4_xxxxxxxxxxxx
GOATX402_API_SECRET=gx4s_xxxxxxxxxxxx

# ERC-8004 Identity
ERC8004_AGENT_ID=<your-agent-id>
```

Save these in `provider-sidecar/.env`:
```env
GOATX402_API_URL=https://x402-api-lx58aabp0r.testnet3.goat.network
GOATX402_MERCHANT_ID=alice-llama3-provider
GOATX402_API_KEY=gx4_xxxxxxxxxxxx
GOATX402_API_SECRET=gx4s_xxxxxxxxxxxx
ERC8004_AGENT_ID=<your-agent-id>
```

The `agentId` will also be included when you register on the marketplace (see SKILL-01, Section 6).

### Step 4: Verify Identity On-Chain

1. Visit [goat-dashboard.vercel.app](https://goat-dashboard.vercel.app)
2. Search for your agent ID or wallet address
3. Confirm your agent profile shows:
   - Project name
   - Description
   - `x402Support: true` flag
   - Correct wallet address

### Step 5: Register on Marketplace with Identity

When registering on the marketplace (via SKILL-01 or manually), include your `agentId`:

```bash
curl -X POST http://localhost:3000/api/providers \
  -H "Content-Type: application/json" \
  -d '{
    "name": "alice-llama3-provider",
    "model": "llama3",
    "endpoint": "http://<your-ip>:4021/v1/chat/completions",
    "pricing": {"amount": "0.01", "symbol": "USDC"},
    "walletAddress": "0xYourWallet",
    "agentId": "<your-agent-id>"
  }'
```

Providers who include a valid `agentId` get a "Verified" badge on their marketplace card, linking to their GOAT dashboard profile.

## Stub Strategy

Before registration completes:
- Providers register on the marketplace without an `agentId` — they simply don't get the "Verified" badge
- Provider sidecar can use `MOCK_PAYMENTS=true` until x402 credentials arrive
- All modules work normally — identity is optional for MVP functionality

## Interface Contract

**Exposes**:
- `agentId` — on-chain identity verifiable at GOAT dashboard (per-provider, stored on Provider object)
- x402 merchant credentials for the provider sidecar

**Consumes**:
- `@goathackbot` registration service

## Integration Points

1. **+ MKT-01**: Provider's `agentId` is stored on the `Provider` object. `ProviderCard` shows a "Verified" badge linking to `https://goat-dashboard.vercel.app/agent/<agentId>`
2. **+ PAY-01**: Provider sets x402 credentials in `provider-sidecar/.env` → sidecar can create/verify real payment orders
3. **+ SKILL-01**: Skill teaches providers to register identity with `@goathackbot` before setting up the sidecar
4. **+ DEMO-01**: Provider's identity is shown in Act 4 of the demo on GOAT dashboard
